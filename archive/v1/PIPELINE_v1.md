Website Builder & Media Assembly Pipeline — Master Documentation
1. Architectural Mandate: The High-End Manual V1 Production Line
This document serves as the absolute operational contract and system directive for the manual assembly of version 1 (V1) websites. It embeds the strict design principles of the MasterClass premium web framework into a predictable programmatic assembly line.

The Production Structure:
Member 1 (Components Builder / Your Role): Engineers the modular, highly responsive, pre-animated layout containers and structural style variations with specific MasterClass phasing parameters.

Member 2 (AI Video Generator): Isolates key product objects and handles start-and-end frame image interpolation to generate high-fidelity cinematic looping video sequences.

Member 3 (The Assembler): Extracts the repository files from Member 1 and the media assets from Member 2, handing them directly to Claude Desktop to orchestrate the final compilation.

Directive for Claude Desktop: You are acting strictly as the Assembly Line Orchestrator. You are forbidden from inventing layout frameworks, writing custom CSS overrides, or altering the internal GSAP timelines built into Member 1's components. Your sole objective is to ingest the extracted code and raw video assets, mapping them seamlessly into the configuration layout tree while strictly tracking the visual hierarchy rules.

2. Core Layout Bricks (The 5-Part Plug-and-Play Story Foundation)
The ./components/ directory houses the functional visual code generated by Member 1. Every website follows a strict pattern: Introduction -> Benefits -> Details -> Final Invitation. Claude Desktop must map incoming target business text and video assets straight to their declared properties (props).

Global Anchor: Navigation.jsx (The Brand Header Entry)

MasterClass Rule: Clean, minimal, and entirely free of distractions.

Mechanics: Sticky blur-glass background pinned to the top of the viewport. Restrained micro-interactions that gracefully shift opacity when hovered.

Prop Signatures: brand_identity_string, navigation_menu_array (Objects: link_label, scroll_anchor_id), action_button_label, motion_tempo.

Chapter 1: HeroSection.jsx (The Arrival Engine)

MasterClass Rule: Sets the mood instantly through high-contrast visual authority and vast micro-white spaces before a visitor reads a single word.

Mechanics: Scroll-linked Z-axis scale-down tunnel inspired by Neverland Agency. Contains an automated, responsive HTML5 video layer backing intended to loop widescreen cinematic content generated by our motion models.

Phasing Blueprint: Text fades in first with an intentional Upper-Glide delay (stagger: 0.15), and media assets arrive last to let the visitor absorb the brand statement without cognitive overload.

Prop Signatures: hook_headline, hook_context, primary_action_label, motion_tempo, depth_layer_one_url (Primary background AI loop video URL), depth_layer_two_url, depth_layer_three_url.

Chapter 2: CollageIntroduction.jsx (The Narrative Layer)

MasterClass Rule: Humanizes the page by establishing personality and storytelling rhythm instead of long, boring copy blocks.

Mechanics: Asymmetric floating canvas matrix inspired by Damai. Images or short video elements move at independent parallax travel velocities relative to scrolling. Uses strict white-space boundaries to separate elements.

Prop Signatures: introStatement, collageImages (Array limited strictly to exactly 4 visual URLs), motion_tempo.

Chapter 3: StaticOfferingsGrid.jsx (The Capability Matrix)

MasterClass Rule: Relies on structural grid symmetry. Avoids cheap, un-engaging repeating sliders or cluttered layouts.

Mechanics: 3-column row architecture inspired by Refokus. Handles pointer-tracking mouse hover coordinates that deforms card borders and smoothly scales monochromatic preview imagery to acknowledge user presence.

Prop Signatures: offering_units (Array slice limited to exactly 3 object pillars containing: item_title, item_summary, item_cover_url), motion_tempo.

Chapter 4: TextEditorial.jsx (The Visual Breathing Room)

MasterClass Rule: Acts as a deliberate pause in conversation, providing an equivalent of digital silence so the visitor's eyes can rest.

Mechanics: Single-column, center-locked minimal typography module strictly enforcing vast top-and-bottom whitespace padding ratios to maximize readability.

Prop Signatures: editorial_header, editorial_body_paragraphs (Array of text strings), motion_tempo.

Chapter 5: TwoColumnValuePause.jsx (The Trust Core / 3D Gateway)

MasterClass Rule: Minimal but not plain, creating an asymmetric balance between bold typography and high-resolution visuals.

Mechanics: Inspired by Quin Global Tax Law. Features an internal Asset Detection Gate: if a sketchfab_embed_uid token is passed, it mounts a clean 3D iframe viewer. If omitted, it gracefully degrades to display Member 2's widescreen showcase video (singular_master_visual).

Prop Signatures: philosophy_anchor, philosophy_deep_dive, singular_master_visual (Accepts fallback showcase asset/video), sketchfab_embed_uid, motion_tempo.

Chapter 6: ClosingCallToAction.jsx (The Immersive Terminal)

MasterClass Rule: Does not overwhelm with options. Leaves a lasting, singular impression and a clear final choice.

Mechanics: Full-screen minimal section inspired by Portman. Backed by a high-performance Three.js WebGL canvas rendering 350 responsive gold micro-particles operating under continuous rotation and scroll-velocity physics.

Prop Signatures: closing_invitation, target_conversion_label, corporate_copyright, direct_contact_strings, motion_tempo.

3. Structural Variation Logic (Claude's Core Task)
To build diverse websites for distinct niches, Claude Desktop must execute structural variations without creating duplicate files or changing the internal code of the layout bricks. Variation is achieved purely via template assembly within the root configuration layer (GeneratedScene.jsx).

Allowed Variation Vectors:

Component Multiplicity & Omission: Depending on the client profile, Claude can choose to omit specific components (e.g., bypassing TwoColumnValuePause if no 3D link or showcase asset is available) or duplicate components (e.g., rendering two separate StaticOfferingsGrid layers back-to-back to expand a large corporate catalog).

Asset Injection & Branding Speed: Claude will alter the entire look and feel by injecting different copy strings, choosing appropriate brand tempos (slow_luxury vs kinetic_modern via motion_registry.js), and changing the media layer backgrounds with Member 2's AI-generated motion loops.

When Member 3 prompts you to assemble a website, follow these steps across your back-and-forth conversational loops:

Step 2: Media Asset Registration: Ingest the explicit video product loops provided by Member 2. Map these URLs directly to the primary background slots (depth_layer_one_url or singular_master_visual).

Step 3: Iterative Composition: Write the single compilation wrapper file: GeneratedScene.jsx. Pass all parsed text assets and product video strings cleanly via props.

Step 4: Refinement Cycles: When Member 3 requests alignment or structural adjustments, modify the arrangement or data arguments inside GeneratedScene.jsx only.
